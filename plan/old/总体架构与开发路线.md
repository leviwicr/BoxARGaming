# 总体架构与开发路线

## 系统概述

"智绘灵境"——基于ESP32-P4的语义增强型AR交互沙箱。核心流程：

```
摄像头拍照 → 目标检测识别物体 → 生成静态游戏地图 → 用户设定终点
→ 物理引擎模拟弹珠运动（受IMU倾斜控制）→ 弹珠到达终点
```

## 关键设计决策

1. **"扫描-游玩"模式**：摄像头仅在初始阶段工作，拍一张照片进行物体检测后，生成固定游戏地图。游戏过程中摄像头不跟踪，弹珠在静态地图上运动，避免设备晃动导致画面与弹珠运动冲突。

2. **坐标系映射**：摄像头画面 (800×640 或按比例) → 游戏地图 (正方形区域) → 屏幕显示 (720×1280 中上方正方形区域)

3. **物理为2D平面模拟**：弹珠在桌面（俯视图）上运动，重力方向由IMU倾斜角度决定。

## 模块架构

```
main.c                          —— 入口，初始化各模块
├── config.h                    —— 全局配置常量
├── camera/camera_driver.c/h    —— MIPI-CSI相机驱动，帧捕获
├── detection/object_detection.c/h —— 目标检测（占位+ESP-DL接口）
├── game/game_map.c/h           —— 游戏地图生成（相机帧+检测结果→静态地图）
├── game/physics_engine.c/h     —— 2D物理引擎（弹珠运动、碰撞）
├── game/game_manager.c/h       —— 游戏状态机、流程控制
├── imu/imu_driver.c/h          —— MPU6050驱动（占位）
└── render/renderer.c/h         —— LVGL渲染（地图背景+弹珠+UI）
```

## 游戏状态机

```
STATE_INIT → STATE_CAPTURE → STATE_DETECT → STATE_MAP_SETUP
→ STATE_SET_TARGET → STATE_PLAYING → STATE_GOAL_REACHED
```

| 状态 | 说明 |
|------|------|
| INIT | 初始化所有硬件和LVGL |
| CAPTURE | 启动相机，捕获一帧画面 |
| DETECT | 运行目标检测，获取物体列表 |
| MAP_SETUP | 在相机帧上绘制检测结果，生成静态游戏地图 |
| SET_TARGET | 等待用户在屏幕上点击设定终点 |
| PLAYING | 物理引擎运行，弹珠运动，实时渲染 |
| GOAL_REACHED | 弹珠到达终点，显示结果，等待重新开始 |

## 开发阶段

按顺序实现，每个阶段完成后编译验证：

### 阶段1：相机驱动与帧捕获
- 集成 MIPI-CSI 摄像头驱动 (OV5647)
- 配置 ISP 进行颜色转换 (RAW8→RGB565)
- 验证能成功捕获帧并打印信息

### 阶段2：目标检测模块（占位实现）
- 实现检测模块接口
- 先用固定数据/简单颜色检测作为占位
- 预留 ESP-DL 模型接口
- 在相机帧上绘制检测框验证

### 阶段3：游戏地图生成与显示
- 将相机帧+检测结果渲染为LVGL画面
- 实现地图坐标系映射
- 显示静态游戏地图在屏幕上

### 阶段4：物理引擎
- 2D弹珠运动模型
- 碰撞检测与响应
- 不同物体的物理属性
- 先用模拟IMU数据测试

### 阶段5：游戏流程控制
- 完整状态机实现
- 触摸设定终点
- 弹珠渲染
- 判定到达终点

### 阶段6：IMU集成（MPU6050）
- I2C驱动
- 卡尔曼滤波
- 倾斜角度到重力的映射

### 阶段7：整体调试与优化
- 各参数调优
- 帧率优化
- ESP-DL真实模型集成（可选）

## 坐标系定义

- **相机坐标系**：以相机帧左上角为原点，x向右，y向下，单位像素
- **游戏地图坐标系**：以地图左上角为原点，x向右，y向下，单位像素。地图为正方形（取相机帧中心最大正方形区域）
- **屏幕坐标系**：LVGL屏幕坐标，x向右，y向下。地图显示在屏幕上方，下方为控制/信息区

```
相机帧 (800×640)
    ┌────────────────────┐
    │                    │
    │   ┌────────┐       │
    │   │ 640×640│       │  ← 取中心正方形作为游戏地图
    │   │ 游戏   │       │
    │   │ 地图   │       │
    │   └────────┘       │
    │                    │
    └────────────────────┘

屏幕 (720×1280 竖屏)
    ┌────────────────────┐
    │                    │
    │   游戏地图区域     │  ← 约680×680，居中
    │   (相机快照+       │
    │    检测框叠加)     │
    │                    │
    ├────────────────────┤
    │   状态/控制区      │  ← 剩余高度(~600px)
    │   倾斜指示器等     │
    └────────────────────┘
```

## 调试策略

1. 每个模块独立可测试
2. 大量使用 ESP_LOGI/ESP_LOGE 输出关键状态
3. 编译时使用 `-O0` 调试，确认无误后切 `-Og` 或 `-Os`
4. 阶段完成后用 idf.py build 验证编译，用户烧录验证实际效果
5. 关键数据结构用 `ESP_LOG_BUFFER_HEXDUMP` 输出

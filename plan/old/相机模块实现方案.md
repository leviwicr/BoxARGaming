# 相机模块实现方案

## 模块目标

通过 MIPI-CSI 接口驱动 OV5647 摄像头，实现：
1. 相机初始化（传感器配置、CSI控制器、ISP）
2. 单帧捕获（拍照模式，不需要连续视频流）
3. 获取 RGB565 格式的帧数据用于后续检测和显示

## 硬件接口

- **接口类型**：MIPI-CSI（2-lane）
- **SCCB/I2C**：用于配置 OV5647 寄存器。根据 Waveshare 原理图确定 SCCB 引脚。查找开发板CSI接口对应的I2C引脚。如GPIO8(SCL)/GPIO7(SDA)被触摸占用，需要确认CSI专用的SCCB引脚或另接
- **MIPI PHY 供电**：ESP32-P4 的 MIPI CSI 需要 2.5V，使用内部LDO供电（通道3）

## 依赖组件

在 `main/idf_component.yml` 中添加：

```yaml
dependencies:
  waveshare/esp32_p4_platform: "^1"
  lvgl/lvgl: "^9"
  espressif/usb: "*"
  espressif/esp_cam_sensor: "*"       # OV5647 传感器驱动
```

ESP-IDF 内置的 `esp_driver_cam` 组件提供 CSI 控制器驱动，无需额外依赖。

## 关键数据结构

```c
// camera_driver.h

#pragma once
#include "esp_err.h"
#include "esp_cam_ctlr.h"
#include "esp_cam_ctlr_csi.h"
#include "driver/isp.h"

#ifdef __cplusplus
extern "C" {
#endif

/**
 * @brief 相机配置结构体
 */
typedef struct {
    uint16_t h_res;         // 水平分辨率
    uint16_t v_res;         // 垂直分辨率
    uint8_t  lane_num;      // MIPI lane 数量 (1 or 2)
    uint32_t lane_bitrate;  // MIPI lane 比特率 (Mbps)，如200
    uint32_t isp_clk_hz;    // ISP 时钟频率，如 80*1000*1000
} camera_config_t;

/**
 * @brief 相机帧数据
 */
typedef struct {
    uint8_t  *buffer;       // 帧数据缓冲区指针
    size_t    buf_len;      // 缓冲区大小（字节）
    uint16_t  width;        // 帧宽度（像素）
    uint16_t  height;       // 帧高度（像素）
} camera_frame_t;

/**
 * @brief 初始化相机
 * @param config 相机配置参数，传NULL使用默认值 (800x640, RGB565)
 * @return ESP_OK 成功，否则失败
 */
esp_err_t camera_init(const camera_config_t *config);

/**
 * @brief 捕获一帧图像
 * @param out_frame 输出帧数据指针，调用者不需要释放buffer
 * @param timeout_ms 超时时间（毫秒）
 * @return ESP_OK 成功
 */
esp_err_t camera_capture_frame(camera_frame_t *out_frame, uint32_t timeout_ms);

/**
 * @brief 反初始化相机，释放资源
 * @return ESP_OK 成功
 */
esp_err_t camera_deinit(void);

/**
 * @brief 获取相机分辨率
 */
void camera_get_resolution(uint16_t *width, uint16_t *height);

#ifdef __cplusplus
}
#endif
```

## 实现步骤

### 1. LDO 供电配置

```c
// 为 MIPI PHY 提供 2.5V
#include "esp_ldo_regulator.h"

static esp_ldo_channel_handle_t ldo_mipi_phy = NULL;

static esp_err_t camera_enable_mipi_ldo(void)
{
    esp_ldo_channel_config_t ldo_config = {
        .chan_id   = 3,            // LDO 通道3
        .voltage_mv = 2500,        // 2.5V
    };
    return esp_ldo_acquire_channel(&ldo_config, &ldo_mipi_phy);
}
```

### 2. 传感器初始化（OV5647）

使用 `esp_cam_sensor` 组件提供的传感器驱动。关键步骤：

```c
#include "esp_cam_sensor.h"

// 传感器配置
// 注意：SCCB I2C引脚需要查阅Waveshare原理图确认
// 如果使用独立的CSI SCCB引脚，需根据实际连接填写
// 假设CSI接口用独立的I2C引脚或与板上I2C共用

// OV5647 需要 MIPI_2lane_24Minput_RAW8_800x640_50fps 这样的配置格式
```

### 3. CSI 控制器初始化

```c
esp_cam_ctlr_csi_config_t csi_config = {
    .ctlr_id                = 0,       // CSI 控制器0
    .h_res                  = 800,
    .v_res                  = 640,
    .lane_bit_rate_mbps     = 200,     // 200Mbps per lane
    .input_data_color_type  = CAM_CTLR_COLOR_RAW8,
    .output_data_color_type = CAM_CTLR_COLOR_RGB565,
    .data_lane_num          = 2,
    .byte_swap_en           = false,
    .queue_items            = 1,       // 单帧模式
};
esp_cam_ctlr_handle_t cam_handle = NULL;
esp_cam_new_csi_ctlr(&csi_config, &cam_handle);
```

### 4. ISP 配置（RAW8→RGB565 转换）

```c
isp_proc_handle_t isp_proc = NULL;
esp_isp_processor_cfg_t isp_config = {
    .clk_hz                 = 80 * 1000 * 1000,
    .input_data_source      = ISP_INPUT_DATA_SOURCE_CSI,
    .input_data_color_type  = ISP_COLOR_RAW8,
    .output_data_color_type = ISP_COLOR_RGB565,
    .has_line_start_packet  = false,
    .has_line_end_packet    = false,
    .h_res                  = 800,
    .v_res                  = 640,
};
esp_isp_new_processor(&isp_config, &isp_proc);
esp_isp_enable(isp_proc);
```

### 5. 帧缓冲区分配

```c
// 帧缓冲区大小计算
size_t frame_size = h_res * v_res * 2;  // RGB565 = 2 bytes/pixel

// 使用 PSRAM 分配（大缓冲区），需对齐到 cache line
size_t alignment = 0;
esp_cache_get_alignment(0, &alignment);
void *frame_buffer = heap_caps_aligned_calloc(alignment, 1, frame_size,
                                               MALLOC_CAP_SPIRAM | MALLOC_CAP_8BIT);
```

### 6. 帧捕获流程

```c
// 注册回调
esp_cam_ctlr_evt_cbs_t cbs = {
    .on_get_new_trans   = on_new_buffer,
    .on_trans_finished  = on_trans_finished,
};
esp_cam_ctlr_register_event_callbacks(cam_handle, &cbs, &trans_data);

// 启动、接收、停止
esp_cam_ctlr_enable(cam_handle);
esp_cam_ctlr_start(cam_handle);
esp_cam_ctlr_receive(cam_handle, &trans_data, timeout_ms);
// 等待帧完成...
esp_cam_ctlr_stop(cam_handle);
esp_cam_ctlr_disable(cam_handle);
```

## 默认配置

| 参数 | 值 | 说明 |
|------|----|------|
| 分辨率 | 800×640 | OV5647 50fps 模式 |
| 输出格式 | RGB565 | 兼容LVGL显示 |
| Lane数 | 2 | MIPI 2-lane |
| Lane速率 | 200 Mbps | 稳定传输速率 |
| ISP时钟 | 80 MHz | ISP处理器时钟 |

## SCCB I2C 引脚说明

**重要：需要确认 OV5647 的 SCCB/I2C 引脚连接。**

Waveshare ESP32-P4-Module-DEV-KIT 的 CSI 接口可能使用以下引脚：
- 方案A：与板载 I2C (GPIO8/7) 共用总线（传感器地址不冲突即可）
- 方案B：独立 SCCB 引脚（需查阅开发板原理图）

实现时需根据实际硬件调整。可以在 `config.h` 中定义：

```c
#define CAMERA_SCCB_SCL_IO   8    // 根据实际修改
#define CAMERA_SCCB_SDA_IO   7    // 根据实际修改
#define CAMERA_SCCB_I2C_PORT I2C_NUM_0  // 或 I2C_NUM_1
```

如果是独立I2C，需要单独初始化一个I2C总线。

## 调试方法

1. **编译后烧录**，通过串口日志确认：
   - LDO 2.5V 供电是否成功
   - OV5647 传感器是否被检测到（读ID寄存器）
   - CSI 控制器初始化是否成功
   - ISP 初始化是否成功
   - 帧捕获是否完成

2. **关键日志输出**：
```c
ESP_LOGI(TAG, "Camera init: %dx%d, %d lanes, %lu Mbps",
         config.h_res, config.v_res, config.lane_num, config.lane_bitrate);
ESP_LOGI(TAG, "Frame captured: %u bytes, first pixels: %02X %02X %02X %02X",
         frame_size, buf[0], buf[1], buf[2], buf[3]);
```

3. **验证帧数据**：输出前几个像素的RGB值，检查是否合理（非全0或全FF）

## 注意事项

1. MIPI CSI 使用 ESP32-P4 专用引脚 42-48，不需要在代码中配置（芯片内部固定连接）
2. OV5647 本身有ISP功能，如果图像质量不满意，可以配合 ESP 片上 ISP 进行调优
3. 如果 SCCB 与触摸共用I2C，注意不能同时操作，需加锁或分时使用
4. 第一次运行时要确认 `esp_cam_sensor` 组件版本支持 OV5647

# 物理引擎模块实现方案

## 模块目标

实现2D平面上的弹珠物理模拟，包括：
1. 重力驱动的加速度（方向和大小由IMU倾斜决定）
2. 速度与位置更新
3. 弹珠与检测到的物体之间的碰撞检测与响应
4. 摩擦力减速
5. 边界约束

## 物理模型

### 坐标系

- 2D笛卡尔坐标系，地图平面
- x轴向右，y轴向下（与屏幕坐标系一致）
- 地图尺寸：MAP_SIZE × MAP_SIZE 像素

### 重力模型

IMU倾斜角度决定重力在地图平面上的分量：

```
实际重力 g = 9.8 m/s²（垂直向下）

设备倾斜后：
  ax = g * sin(θx)    ← x方向加速度分量（左右倾斜）
  ay = g * sin(θy)    ← y方向加速度分量（前后倾斜）

映射到地图坐标系（像素/s²）：
  gravity_x = ax * PIXELS_PER_METER    ← 像素加速度
  gravity_y = ay * PIXELS_PER_METER
```

`PIXELS_PER_METER` 需要标定：它表示实际1米对应地图上多少像素。
例如：桌面宽约0.6m，地图640px → `PIXELS_PER_METER ≈ 1067`

### 弹珠运动方程（Euler积分）

```c
每帧更新 (dt = 1/PHYSICS_HZ)：

// 加速度 = 重力分量
accel_x = gravity_x
accel_y = gravity_y

// 速度更新（含摩擦力减速）
vel_x += accel_x * dt
vel_y += accel_y * dt

// 摩擦力（与速度方向相反）
speed = sqrt(vel_x² + vel_y²)
if (speed > 0) {
    friction_force = friction_coeff * g_normal * dt
    speed = max(0, speed - friction_force)
    vel_x *= speed / old_speed
    vel_y *= speed / old_speed
}

// 位置更新
pos_x += vel_x * dt + 0.5 * accel_x * dt²
pos_y += vel_y * dt + 0.5 * accel_y * dt²
```

## 碰撞检测

### 物体简化模型

将检测到的物体简化为圆形或矩形碰撞体：

```c
typedef enum { COLLISION_CIRCLE, COLLISION_RECT } collision_shape_t;

typedef struct {
    collision_shape_t shape;
    union {
        struct { float cx, cy, radius; } circle;
        struct { float x, y, w, h; } rect;
    };
    float bounce;      // 反弹系数
    float friction;    // 摩擦系数
} collision_body_t;
```

### 弹珠-圆形碰撞

```c
static bool check_circle_collision(float mx, float my, float mr,
                                    float cx, float cy, float cr,
                                    float *nx, float *ny, float *penetration)
{
    float dx = mx - cx;
    float dy = my - cy;
    float dist = sqrtf(dx * dx + dy * dy);
    float min_dist = mr + cr;

    if (dist >= min_dist || dist < 0.001f) return false;

    *nx = dx / dist;       // 碰撞法线方向（从圆心指向弹珠）
    *ny = dy / dist;
    *penetration = min_dist - dist;  // 穿透深度
    return true;
}
```

### 弹珠-矩形碰撞

将矩形分解为4条边，检测弹珠与边的碰撞；同时检测弹珠是否在矩形内部或角点附近。

简化方案：将矩形扩展弹珠半径，检测弹珠中心点是否在扩展后的矩形内，如果在，则弹珠与矩形碰撞。

```c
static bool check_rect_collision(float mx, float my, float mr,
                                  float rx, float ry, float rw, float rh,
                                  float *nx, float *ny, float *penetration)
{
    // 找到矩形上距离弹珠中心最近的点
    float closest_x = fmaxf(rx, fminf(mx, rx + rw));
    float closest_y = fmaxf(ry, fminf(my, ry + rh));

    float dx = mx - closest_x;
    float dy = my - closest_y;
    float dist_sq = dx * dx + dy * dy;

    if (dist_sq >= mr * mr) return false;

    float dist = sqrtf(dist_sq);
    if (dist < 0.001f) {
        // 弹珠中心在矩形内部，向最近的边推出
        float left = mx - rx;
        float right = rx + rw - mx;
        float top = my - ry;
        float bottom = ry + rh - my;
        float min_edge = fminf(fminf(left, right), fminf(top, bottom));
        if (min_edge == left)      { *nx = -1; *ny = 0; }
        else if (min_edge == right) { *nx = 1; *ny = 0; }
        else if (min_edge == top)   { *nx = 0; *ny = -1; }
        else                       { *nx = 0; *ny = 1; }
        *penetration = mr;
    } else {
        *nx = dx / dist;
        *ny = dy / dist;
        *penetration = mr - dist;
    }
    return true;
}
```

### 碰撞响应

```c
static void resolve_collision(marble_physics_t *marble,
                               float nx, float ny, float penetration,
                               float bounce_coeff)
{
    // 1. 位置修正：将弹珠推出穿透
    marble->pos_x += nx * penetration;
    marble->pos_y += ny * penetration;

    // 2. 速度响应
    // 将速度分解为法向和切向分量
    float vn = marble->vel_x * nx + marble->vel_y * ny;  // 法向分量
    float vt_x = marble->vel_x - vn * nx;
    float vt_y = marble->vel_y - vn * ny;

    // 仅反向法向分量（反弹），切向分量保留
    if (vn < 0) {  // 只在弹珠向物体运动时才反弹
        marble->vel_x = vt_x - bounce_coeff * vn * nx;
        marble->vel_y = vt_y - bounce_coeff * vn * ny;
    }
}
```

## 物理更新主循环

```c
// 在 game_manager 中以固定频率调用
void physics_update_task(void)
{
    TickType_t last_wake = xTaskGetTickCount();
    const TickType_t period = pdMS_TO_TICKS(1000 / PHYSICS_HZ);

    while (1) {
        // 1. 读取IMU数据，设置重力
        float ax, ay;
        imu_get_tilt_accel(&ax, &ay);
        physics_set_gravity(ax, ay);

        // 2. 执行物理步进
        const marble_physics_t *marble = physics_step(1.0f / PHYSICS_HZ);

        // 3. 更新渲染
        marble_render_info_t render_info = {
            .map_x = marble->pos_x,
            .map_y = marble->pos_y,
            .radius = marble->radius
        };
        renderer_update_marble(&render_info);

        // 4. 检测是否到达目标
        if (physics_check_goal_reached(target_x, target_y, THRESHOLD)) {
            // 触发状态切换
        }

        vTaskDelayUntil(&last_wake, period);
    }
}
```

## 注意事项

1. **时间步长**：使用固定dt，不依赖实际帧率，确保物理确定性
2. **碰撞优先级**：如果弹珠同时与多个物体碰撞，按穿透深度排序，优先解决最深的
3. **速度限制**：设置最大速度防止弹珠"穿过"物体（隧道效应）
4. **静止判定**：当速度低于阈值时，将速度归零，避免微小滑动
5. **物理参数可调**：将所有参数放在 `config.h` 中，方便调优

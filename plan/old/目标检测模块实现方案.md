# 目标检测模块实现方案

## 模块目标

对相机捕获的帧进行目标检测，识别桌面上的物体（水杯、橡皮、书本等），输出每个物体的：
- 类别标签
- 包围框（bounding box）
- 中心点坐标
- 物理属性（反弹系数、摩擦系数）

## 设计策略

**采用两阶段设计：**

1. **占位实现（Phase 1）**——用简单的颜色/形状检测或固定数据，快速打通整个系统流程，便于调试其他模块
2. **ESP-DL模型集成（Phase 2）**——训练量化后的目标检测模型，替换占位实现

这样设计的好处：物理引擎、渲染、游戏流程等模块可以先使用占位检测数据开发调试，不阻塞整体进度。

## 接口定义

```c
// object_detection.h

#pragma once
#include "esp_err.h"
#include <stdint.h>

#ifdef __cplusplus
extern "C" {
#endif

/** 支持的物体类别 */
typedef enum {
    DETECT_CLASS_NONE = 0,
    DETECT_CLASS_CUP,        // 水杯（陶瓷/塑料）
    DETECT_CLASS_ERASER,     // 橡皮
    DETECT_CLASS_BOOK,       // 书本
    DETECT_CLASS_PEN,        // 笔
    DETECT_CLASS_OTHER,      // 其他物体
    DETECT_CLASS_MAX
} detect_class_t;

/** 物体物理属性 */
typedef struct {
    float bounce_coeff;      // 反弹系数 0.0~1.0
    float friction_coeff;    // 摩擦系数 0.0~1.0
} object_physics_t;

/** 检测到的单个物体 */
typedef struct {
    detect_class_t class_id;       // 类别
    char name[16];                 // 名称
    uint16_t x, y;                 // 包围框左上角 (像素坐标)
    uint16_t w, h;                 // 包围框宽高
    uint16_t cx, cy;               // 中心点坐标
    object_physics_t physics;      // 物理属性
} detected_object_t;

/** 检测结果 */
typedef struct {
    detected_object_t *objects;    // 物体数组（动态分配）
    uint8_t count;                 // 物体数量
    uint16_t frame_width;          // 原帧宽度
    uint16_t frame_height;         // 原帧高度
} detection_result_t;

/**
 * @brief 初始化检测模块
 * @note 占位模式只需初始化数据结构
 *       ESP-DL模式需要加载模型文件
 * @return ESP_OK 成功
 */
esp_err_t detection_init(void);

/**
 * @brief 对一帧图像执行目标检测
 * @param frame_data   RGB565格式帧数据
 * @param frame_width  帧宽度
 * @param frame_height 帧高度
 * @param result       输出检测结果，调用者需要用 detection_free_result 释放
 * @return ESP_OK 成功
 */
esp_err_t detection_run(const uint8_t *frame_data,
                        uint16_t frame_width, uint16_t frame_height,
                        detection_result_t *result);

/**
 * @brief 释放检测结果内存
 */
void detection_free_result(detection_result_t *result);

/**
 * @brief 反初始化检测模块
 */
void detection_deinit(void);

#ifdef __cplusplus
}
#endif
```

## 物理属性映射表

| 物体类别 | 反弹系数 | 摩擦系数 | 说明 |
|---------|---------|---------|------|
| 水杯（陶瓷/硬质） | 0.7 | 0.15 | 硬表面，弹珠弹跳明显 |
| 橡皮 | 0.25 | 0.8 | 软质高摩擦，弹珠几乎不反弹 |
| 书本 | 0.35 | 0.55 | 纸质，中等弹性和摩擦 |
| 笔 | 0.6 | 0.2 | 硬质光滑 |
| 其他/未知 | 0.5 | 0.4 | 默认中等值 |

这些值需要在实际上测试中调试，目标让交互体感"直觉正确"。

## 占位实现方案

占位检测用于开发调试，提供三种模式（通过宏切换）：

### 模式A：固定测试数据
在代码中硬编码若干物体，用于最基础的流程测试。

### 模式B：简单颜色阈值检测
将 RGB565 帧转换到 HSV 空间，用颜色范围检测物体：
- 白色/浅色区域 → 可能是水杯/纸张
- 深色区域 → 可能是书本
- 彩色（如粉红/蓝色）区域 → 可能是橡皮

```c
// 颜色检测辅助函数
static void rgb565_to_hsv(uint16_t rgb, float *h, float *s, float *v);

// 对每个颜色范围扫描图像，找到连续区域
// 简单的滑动窗口 + 颜色匹配 + 连通区域标记
static uint8_t detect_by_color(const uint8_t *frame, uint16_t w, uint16_t h,
                                detected_object_t *objects, uint8_t max_objects);
```

### 模式C：轮廓检测
1. RGB565 → 灰度图
2. Canny 边缘检测（简化版）
3. 查找轮廓
4. 用矩形包围盒近似

轮廓检测计算量较大，但比颜色更通用。

**推荐使用模式A+B混合**：先用固定数据验证流程，再用简单颜色检测作为半真实验证。

```c
// 编译选择：根据宏切换检测方式
#define DETECTION_MODE  1  // 0=固定数据, 1=颜色检测, 2=轮廓检测(未来), 3=ESP-DL(未来)
```

## ESP-DL 模型集成方案（Phase 2）

### 模型选择

推荐使用 YOLO 系列轻量模型，ESP-DL 已有成熟支持：
- **YOLO11n**：约2.6M参数，在ESP32-P4上推理约200-500ms
- **ESPDet-Pico**：更轻量，约0.5M参数，推理更快约50-100ms
- 自定义训练的轻量检测模型

### 模型准备流程（在PC上完成）

1. **收集数据集**：拍摄桌面物体照片（建议500+张），标注包围框和类别
2. **训练模型**：使用 esp-detection 或 ultralytics 训练
3. **导出 ONNX**：训练好的模型导出为 ONNX 格式
4. **量化**：使用 ESP-PPQ 工具将 FP32 模型量化为 INT8 `.espdl` 格式
5. **部署**：将 `.espdl` 文件放入 SPIFFS 分区或嵌入固件

### ESP-DL 推理接口

```c
// ESP-DL 模式下的推理流程
#if DETECTION_MODE == 3
#include "dl_model_base.hpp"

static void *dl_model = NULL;  // ESP-DL 模型句柄

esp_err_t detection_run(const uint8_t *frame_data, ...)
{
    // 1. 预处理：缩放帧到模型输入尺寸（如320×320）
    preprocess_frame(frame_data, width, height, model_input);

    // 2. 推理
    dl_model->run(model_input);

    // 3. 后处理：解析输出张量，获得包围框
    parse_detection_output(dl_model->get_output(), result);
}
#endif
```

### 预处理流水线

```
RGB565帧 (800×640)
  → 缩放到模型输入尺寸 (如320×320)
  → 归一化 (scale to [0,1] or [-1,1])
  → NCHW 格式布局转换
  → 输入模型
```

### 后处理（NMS非极大值抑制）

```
模型输出 → 解码包围框 (cx,cy,w,h,conf,class_probs)
  → 过滤低置信度 (< 0.5)
  → NMS去重 (IoU > 0.45 的框合并)
  → 输出最终检测结果
```

## 调试方法

1. **占位模式**：
   - 输出检测到的物体信息：`ESP_LOGI("detect", "Found %s at (%d,%d) size %dx%d", obj.name, obj.x, obj.y, obj.w, obj.h);`
   - 在渲染模块中绘制检测框，肉眼验证位置是否合理

2. **ESP-DL模式**：
   - 输出模型加载状态
   - 输出推理耗时（用于性能评估）
   - 用 `ESP_LOG_BUFFER_HEXDUMP` 输出模型输出的前若干字节

3. **切换模式**：在 `config.h` 中修改 `DETECTION_MODE` 宏即可切换

## config.h 中的相关配置

```c
// ========== 检测模块配置 ==========
#define DETECTION_MODE  1        // 检测模式: 0=固定数据, 1=颜色, 3=ESP-DL
#define DETECTION_MAX_OBJECTS  10  // 最多检测物体数
#define DETECTION_CONFIDENCE_THRESHOLD  0.5f  // 置信度阈值(ESP-DL模式)
#define DETECTION_NMS_IOU_THRESHOLD      0.45f // NMS IoU阈值
```

## 注意事项

1. 占位模式和ESP-DL模式使用相同的接口，切换时其他模块不需要修改
2. 颜色检测假设光照条件相对稳定，实际桌面光照变化会影响效果
3. ESP-DL 模型文件（.espdl）最大建议不超过 4MB，需要考虑存储
4. 推理期间可能阻塞其他任务，建议在独立任务中运行

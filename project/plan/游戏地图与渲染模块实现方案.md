# 游戏地图与渲染模块实现方案

## 模块概述

包含两个紧密关联的子模块：
1. **游戏地图模块** (`game_map.c/h`)——负责将相机帧和检测结果合成为静态游戏地图
2. **渲染模块** (`renderer.c/h`)——负责LVGL显示管理（游戏地图背景 + 弹珠 + UI元素）

## 一、游戏地图模块

### 核心任务

输入：相机帧（RGB565, 800×640）+ 检测结果（物体列表）
输出：游戏地图（RGB565正方形图, 640×640）+ 物体在地图坐标系中的位置

### 坐标映射

```
相机帧 (800×640) → 裁剪中心正方形 (640×640) → 缩放到游戏地图 (MAP_SIZE×MAP_SIZE)
                                                        ↓
                                                  映射到屏幕坐标
                                                        ↓
                                                  创建LVGL图像
```

### 接口定义

```c
// game_map.h

#pragma once
#include "esp_err.h"
#include "detection/object_detection.h"
#include <stdint.h>

#ifdef __cplusplus
extern "C" {
#endif

/** 游戏地图中的物体（地图坐标系） */
typedef struct {
    detect_class_t class_id;
    char name[16];
    float map_x, map_y;        // 地图坐标（归一化0.0~1.0或像素坐标）
    float map_w, map_h;        // 地图坐标下的宽高
    object_physics_t physics;
} map_object_t;

/** 游戏地图 */
typedef struct {
    uint16_t width;            // 地图宽度（像素）
    uint16_t height;           // 地图高度（像素）
    uint8_t *image_data;       // 地图图像数据（RGB565），动态分配
    size_t   image_size;       // 图像数据大小（字节）
    map_object_t *objects;     // 地图中的物体
    uint8_t  object_count;     // 物体数量
    uint16_t target_x;         // 目标点X（地图坐标），0表示未设置
    uint16_t target_y;         // 目标点Y（地图坐标）
} game_map_t;

/**
 * @brief 从相机帧和检测结果生成游戏地图
 * @param frame_data    相机原始帧（RGB565）
 * @param frame_w       帧宽度
 * @param frame_h       帧高度
 * @param det_result    检测结果
 * @param map           输出游戏地图
 * @return ESP_OK
 */
esp_err_t game_map_generate(const uint8_t *frame_data,
                            uint16_t frame_w, uint16_t frame_h,
                            const detection_result_t *det_result,
                            game_map_t *map);

/**
 * @brief 将相机坐标转换为地图坐标
 */
void game_map_camera_to_map(uint16_t cam_x, uint16_t cam_y,
                            uint16_t cam_w, uint16_t cam_h,
                            uint16_t map_w, uint16_t map_h,
                            uint16_t *map_x, uint16_t *map_y);

/**
 * @brief 释放游戏地图资源
 */
void game_map_free(game_map_t *map);

/**
 * @brief 设置目标点位置（地图坐标）
 */
void game_map_set_target(game_map_t *map, uint16_t x, uint16_t y);

#ifdef __cplusplus
}
#endif
```

### 地图生成流程

```
步骤1: 从相机帧中心裁剪正方形区域
    crop_x = (800 - 640) / 2 = 80
    crop_y = 0 (640高居中, 640=640不需要裁剪y)
    裁剪区域: (80, 0) → (720, 640)

步骤2: 将裁剪区域缩放到 MAP_SIZE × MAP_SIZE
    使用双线性插值或最近邻缩放

步骤3: 在缩放后的图像上绘制检测框和标签
    检测框坐标系也需同步转换

步骤4: 将物体位置转换为地图坐标系
    map_x = (cam_cx - crop_x) * MAP_SIZE / 640
    map_y = cam_cy * MAP_SIZE / 640
```

## 二、渲染模块

### 核心任务

使用 LVGL 将游戏地图、弹珠、UI 元素渲染到屏幕上。

### 屏幕布局

```
720×1280 (竖屏)
┌──────────────────────┐ y=0
│                      │
│    ┌──────────────┐  │
│    │              │  │
│    │  游戏地图    │  │  640×640 (或适配宽度)
│    │  LV Image    │  │  居中放置
│    │              │  │
│    └──────────────┘  │
│                      │
│  ┌─ 状态信息 ──────┐ │
│  │ 模式: 设置目标  │ │
│  │ 物体数: 3       │ │
│  │ 倾斜角: X Y     │ │  ~600px 信息/控制区域
│  │                 │ │
│  │  [重新开始]     │ │
│  └─────────────────┘ │
└──────────────────────┘
```

### 接口定义

```c
// renderer.h

#pragma once
#include "lvgl.h"
#include "game/game_map.h"
#include <stdint.h>

#ifdef __cplusplus
extern "C" {
#endif

/** 渲染器配置 */
typedef struct {
    uint16_t screen_w;

    uint16_t map_display_y;      // 地图在屏幕上的Y偏移

/** 弹珠渲染信息 */
typedef struct {
    float map_x, map_y;          // 弹珠在地图坐标中的位置
    float radius;                // 弹珠半径（地图坐标像素）
} marble_render_info_t;

/**
 * @brief 初始化渲染器
 * @param cfg 渲染配置
 */
void renderer_init(const renderer_config_t *cfg);

/**

 */

 */

/**
 * @brief 更新目标标记

 * @brief 更新状态栏文本

/**
 * @brief 将屏幕触摸坐标转换为地图坐标

#ifdef __cplusplus
}

```

### LVGL对象层级

```
scr (screen)
├── map_container (固定位置容器)
│   └── map_image (lv_image, 显示游戏地图)
├── marble_obj (lv_obj, 圆形，弹珠，浮动在地图上方)
├── target_marker (lv_obj, 十字标记，浮动在地图上方)
├── status_label (lv_label, 状态文字)
├── info_label (lv_label, 信息文字)
├── tilt_indicator (lv_obj, 倾斜方向指示器)
└── restart_btn (lv_btn, 重新开始按钮)
```

### 弹珠渲染实现

弹珠使用 LVGL 样式实现（不用图片，节省内存）：

```c
// 创建弹珠对象
lv_obj_t *marble = lv_obj_create(map_container);
lv_obj_set_size(marble, MARBLE_RADIUS * 2, MARBLE_RADIUS * 2);
lv_obj_set_style_radius(marble, LV_RADIUS_CIRCLE, 0);

// 渐变效果（模拟3D弹珠）
lv_obj_set_style_bg_color(marble, lv_color_hex(0x4488FF), 0);
lv_obj_set_style_bg_grad_color(marble, lv_color_hex(0x001144), 0);
lv_obj_set_style_bg_grad_dir(marble, LV_GRAD_DIR_VER, 0);

// 阴影
lv_obj_set_style_shadow_color(marble, lv_color_hex(0x000000), 0);
lv_obj_set_style_shadow_width(marble, 8, 0);
lv_obj_set_style_shadow_ofs_x(marble, 3, 0);
lv_obj_set_style_shadow_ofs_y(marble, 3, 0);
lv_obj_set_style_shadow_opa(marble, LV_OPA_40, 0);

// 设置为最上层
lv_obj_move_foreground(marble);

// 更新弹珠位置
void renderer_update_marble(const marble_render_info_t *marble_info)
{
    // 地图坐标 → 屏幕坐标
    lv_coord_t screen_x = map_display_x + (lv_coord_t)(marble_info->map_x);
    lv_coord_t screen_y = map_display_y + (lv_coord_t)(marble_info->map_y);
    lv_obj_set_pos(marble_obj, screen_x - MARBLE_RADIUS, screen_y - MARBLE_RADIUS);
}
```

### 地图坐标 ↔ 屏幕坐标 转换

```c
// 地图坐标 → 屏幕坐标（LVGL）
static inline lv_coord_t map_to_screen_x(float map_x) {
    return map_display_x + (lv_coord_t)(map_x);
}
static inline lv_coord_t map_to_screen_y(float map_y) {
    return map_display_y + (lv_coord_t)(map_y);
}

// 屏幕坐标 → 地图坐标（触摸事件）
bool renderer_screen_to_map_coord(lv_point_t *screen_point,
                                   uint16_t *map_x, uint16_t *map_y)
{
    int rel_x = screen_point->x - map_display_x;
    int rel_y = screen_point->y - map_display_y;
    if (rel_x < 0 || rel_x >= map_display_size ||
        rel_y < 0 || rel_y >= map_display_size) {
        return false;  // 点击在地图外
    }
    // 屏幕显示尺寸 → 地图实际尺寸
    *map_x = rel_x * map_actual_size / map_display_size;
    *map_y = rel_y * map_actual_size / map_display_size;
    return true;
}
```

## 渲染帧率优化

1. **背景地图**只在生成时设置一次，不重复绘制
2. **弹珠**使用 LVGL 对象移动，不重绘整个地图
3. **目标标记**同理，只在设置/变化时更新
4. 启用 LVGL 的 `LV_USE_DRAW_PPA` 使用硬件加速
5. 考虑使用 `lv_refr_now()` 手动刷新，减少延迟

## 调试方法

1. **验证地图生成**：在生成的地图图像中写入测试图案（如四角标记），在屏幕上显示检查裁剪/缩放是否正确

2. **验证坐标转换**：
```c
// 测试：在四个角和中心放置标记点
draw_crosshair_rgb565(map_data, MAP_SIZE, MAP_SIZE, 0, 0, 10, 0xFF, 0, 0);
draw_crosshair_rgb565(map_data, MAP_SIZE, MAP_SIZE, MAP_SIZE-1, 0, 10, 0xFF, 0, 0);
// ... 四个角
draw_crosshair_rgb565(map_data, MAP_SIZE, MAP_SIZE, MAP_SIZE/2, MAP_SIZE/2, 10, 0, 0xFF, 0);
```

3. **验证弹珠渲染**：设定弹珠在已知位置，检查屏幕上位置是否正确

4. **帧率监控**：在状态栏显示当前LVGL刷新帧率
